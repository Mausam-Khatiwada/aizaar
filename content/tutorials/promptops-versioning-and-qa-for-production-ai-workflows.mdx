## QuickAnswer
Treat prompts as production assets, not chat messages. Once prompts drive client-facing output, you need version control, evaluation criteria, owner sign-off, and rollback capability.

## PromptOps principles
- prompts are assets
- assets need ownership
- changes need review
- quality needs measurement
- production failures need rollback

Without this, teams ship unstable output and spend time firefighting.

## Prompt artifact template
For each critical prompt, track:
- use case
- input constraints
- expected output format
- accepted failure modes
- owner
- last validation date
- linked examples (good and bad)

This can live in [Notion AI](/tools/notion-ai) or repo docs.

## Versioning model
Use semantic prompt versions:
- `v1.0.0` baseline prompt
- `v1.1.0` behavior expansion
- `v1.1.1` bug fix wording
- `v2.0.0` structural rewrite

Never silently edit production prompts. If behavior changes, version changes.

## QA scoring rubric
Define objective checks:
1. format compliance
2. factual reliability
3. tone and brand fit
4. actionability
5. latency/cost

Score every candidate prompt change across the same test set.

## Evaluation workflow (practical)
1. Draft prompt change in sandbox.
2. Run 20 representative test cases.
3. Compare old vs new outputs side-by-side.
4. Mark regressions by severity.
5. Approve or reject with reason.

Use [ChatGPT](/tools/chatgpt) or [Claude](/tools/claude) as evaluators for structured grading, but keep final approval human.

## Rollback and incident handling
When output quality drops:
- freeze new prompt deploys
- revert to last stable prompt version
- run incident postmortem
- add regression case to test set

If your workflow has no rollback path, it is not production-ready.

## Cost control by prompt design
PromptOps is also FinOps:
- shorter system instructions where possible
- avoid redundant constraints
- move static context to reusable references
- cache common inputs when architecture allows

Track cost per accepted output, not cost per generation.

## South Asian production context
For multilingual campaigns, keep localized prompt variants:
- English baseline
- Hinglish variant
- Nepali brand voice variant

Do not auto-translate brand-critical copy without review.

## FAQs
### Do we need a dedicated PromptOps engineer?
Not always. Start with one workflow owner and a strict review checklist.

### Can we store prompts only inside chat tools?
For exploration yes. For production no, because discoverability and audit trails are weak.

### How often should prompts be revalidated?
At least monthly, and immediately after major model behavior changes.

### Is PromptOps only for large teams?
No. Small teams benefit quickly because bad outputs cost them disproportionately.

### What metric best reflects PromptOps quality?
Accepted output rate with low revision count.
